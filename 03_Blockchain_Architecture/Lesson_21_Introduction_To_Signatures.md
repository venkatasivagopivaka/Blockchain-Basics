### Lesson 21: An Introduction to Cryptographic Signatures

**1. Core Concept**

The security of a blockchain relies entirely on public key cryptography, not passwords. When you send a transaction, you are effectively placing a tamper-proof digital "wax seal" on a message. This mathematical signature allows the entire decentralized network to instantly verify two things: **Authenticity** (the transaction genuinely came from your wallet) and **Integrity** (the transaction details—like the amount and destination—were not altered in transit).

**2. Key Mechanisms**
* **Seed Phrase (Mnemonic):** The 12- or 24-word master key of your wallet. From this single phrase, an infinite number of private keys can be deterministically generated.
* **Private Key:** A secret cryptographic string used exclusively to sign messages and authorize transactions. If this is exposed, your funds are permanently compromised.
* **Public Key (Asymmetric Cryptography):** Derived from the private key via a mathematically irreversible one-way function (specifically, Elliptic Curve Cryptography). It is shared publicly so the network can verify your signatures.
* **Wallet Address:** A shorter, user-friendly version of the public key. On Ethereum, it is generated by taking the Keccak-256 hash of the public key and keeping the last 20 bytes (prefixed with `0x`).
* **Digital Signature (ECDSA):** The unique mathematical output created when a private key signs a specific message. If even one character of the original message is changed, the signature completely breaks, and the network will reject the transaction.

**3. The "Why It Matters" (Security Perspective)**

As an auditor, understanding signatures is critical because smart contracts often verify them manually using built-in functions like `ecrecover`. If a developer implements signature verification incorrectly, it can lead to catastrophic exploits. Two common vulnerabilities are **Signature Replay Attacks** (where an attacker takes a valid signature used on one contract and reuses it on another) and **Signature Malleability** (where an attacker mathematically tweaks a valid signature to look like a brand new one, tricking the contract into processing the same transaction twice). Always treat user-provided signatures as highly dangerous input that must be strictly validated.

**4. Terms to Remember**
* **Public Key Cryptography (Asymmetric Cryptography):** A cryptographic system that uses pairs of keys: public keys (which may be known to others) and private keys (which may never be known by any except the owner).
* **ECDSA (Elliptic Curve Digital Signature Algorithm):** The specific cryptographic algorithm used by Ethereum and Bitcoin to ensure that funds can only be spent by their rightful owners.
* **Integrity:** The cryptographic guarantee that a payload or message has not been intercepted and modified.
* **Authenticity:** The cryptographic guarantee that a payload or message originated from a specific, verified sender.
